Structura proiectului:

* folder  "C implementations" :  contine implementarile in limbajul C ale celoi doi algoritmi alesi pentru rezolvarea problemei, generatorul de date, precum si datele de intrare si iesire pentru cei doi algortimi (folder "data");

* folder "Py implementations" :  contine implementarile in limbajul Python ale celor doi algoritmi cat si a generatorului de date, precum si datele de intrare-iesire aferente celor doi algoritmi (folder "data") (mentiune : in cazul implementarii algoritmului lui Kruskal in limbajul Python exista un singur set de date de intrare intrucat algoritmul nu returneaza date viabile(mai multe detalii in raportul tehnic) );

* fisier "raport_tehnic" - contine fisierul PDF-ul aferent raportului tehnic, precum si fisierul sursa .tex al acestuia; fisierul contine si sursa raportului tehnic generata de editorul online Overleaf; 

 ** Pentru rularea unui program in limbajul C, respectiv in limbajul Python, se vor introduce la consola numarul de teste care se doresc efectuate(in cazul algoritmului lui Prim, atat in limbajul C cat si in limbajul Python, s-au efectuat teste pe maxim 10 seturi de date; in cazul algoritmului lui Kruskal, numarul de teste efectuate in limbajul C este de maxim 5, datorita unor factori specificati in fisierul .txt atasat folderului aferent).Rezultatele pot fi vizualizate in fisierele .out atasate fiecarui folder-ului fiecarui algoritm, respectiv, fiecarei implementari.
